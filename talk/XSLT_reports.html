XSLT

Extensible Stylesheet Language Transformations

Transforms an XML file into a different file

XML

<specimen>
    <prefix>I</material>
    <regno>1234</colour>
    <collection>Entomology</texture>
    <identification>Camponotus</identification>
    <location>Australia</location>
</specimen>

XML -> XML

<specimen>
    <prefix>I</material>
    <regno>1234</colour>
    <collection>Entomology</texture>
    <identification>Camponotus</identification>
    <location>Australia</location>
</specimen>
<record>
    <entomology>
        <type>specimen</type>
        <registration>I1234<registration>
        <id>Camponotus</id>
        <collected>Australia</collected>
    </entomology>
</record>

Pros

  • Already built into EMu
  • Old - lots of support on internet!
  • Extremely varied output

Cons

  • initially confusing
    • Verbose at some times
    • Terse at others
  • useless error messages from built-in processor
  • Can get slow > c.50,000 bits of data
    • (5,000 records with 10 fields)

General structure of XSLT report file

<stylesheet>
   
    <Output_file> specified file type XML/HTML/text

      - CSV headers, HTML, XML, javascript etc.
        
        <xsl:apply-templates select="/table/tuple" />
        
      - Anything else required for the output

    </Output_file>

    <xsl:template>
        The output for a single record
    </xsl:template>

</stylesheet>

CSV file

<stylesheet>

    <Output_file> is a text file

       "field 1", "field 2", "field 3"

        <xsl:apply-templates select="/table/tuple" />
    
    </Output_file>

    <xsl:template >
        field 1, field 2, field 3
    </xsl:template>

</stylesheet>

KML XML file


<stylesheet>
   
    <Output_file> is an XML file

        KML stuff

        Style choices

        <xsl:apply-templates select="/table/tuple" />
        
        more KML stuff

    </Output_file>

    <xsl:template>
         XML for each record
            if coordinates: style 1
            if no-coordinates: style 2
            if no locality: style 3
    </xsl:template>

</stylesheet>

CSV import file

<stylesheet>
   
    <Output_file> is a text file 

           "irn", "import_field1", "import_field2" [...] 
        
        <xsl:apply-templates select="/table/tuple" />
        
    </Output_file>

    <xsl:template match ="/table/tuple">
        <xsl:if test="comparison of two fields">
           irn, import_field1, import_field2 [...] 
        </xsl:if>
    </xsl:template>

</stylesheet>
